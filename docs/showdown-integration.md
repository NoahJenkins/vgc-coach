# Showdown Integration

Pokemon Showdown should be treated as a testing and review surface first.

## V1 Supports

- exporting team drafts into Showdown-friendly text
- designing structured matchup test plans
- reviewing replays, logs, and manual battle summaries
- turning replay lessons into team or piloting adjustments

## V1 Does Not Support

- autonomous ladder play
- autonomous set testing that claims authoritative matchup truth
- battle simulation used as a replacement for human review

## Why

The project should not recreate the optimizer problem by inventing confidence before the battle-state and evaluation layers are trustworthy.

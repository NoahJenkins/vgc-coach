# Output Rubric

Score each response on:

- `Decision framing`
  - states what the calc is trying to decide
- `Input discipline`
  - lists known assumptions and missing pieces clearly
- `Benchmark selection`
  - focuses on the few interactions that matter most
- `Practical translation`
  - explains what the result changes in building or play
- `Honesty`
  - avoids fake exactness when inputs are incomplete
- `Exact evidence`
  - uses backend/site/assumption details when an exact browser calc succeeded
  - clearly marks fallback or blocked exact runs instead of silently implying verification

Failure triggers:

- invented exact numbers
- exact-browser output with no backend or assumption framing
- no list of assumptions
- too many irrelevant benchmarks
- no explanation of what the result changes

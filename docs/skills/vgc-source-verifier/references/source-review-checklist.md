# Source Review Checklist

## Define The Claim

Before judging any source, answer:

- What exact statement is being audited?
- Is it a rules claim, usage claim, team-trend claim, or matchup claim?
- Does the claim include extra certainty or time language not present in the source?

## Source Review Order

- official rules or event-policy sources for legality and format claims
- dated community sources for usage, cores, and tournament trends
- repo snapshot artifacts for support or fallback context

## Support Strength

Use these buckets:

- `supported` when the source directly justifies the full claim
- `partially supported` when the source backs only part of the claim
- `stale` when the source was once relevant but is no longer safe as present-tense proof
- `conflicted` when credible sources disagree
- `unsupported` when the source does not justify the claim

## Avoid These Failure Modes

- reading the conclusion into the source
- confusing a sample team with universal metagame proof
- turning soft interpretation into direct citation
- ignoring dates, scope, or source rank

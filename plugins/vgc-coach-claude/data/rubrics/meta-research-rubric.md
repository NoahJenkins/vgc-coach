# Meta Research Rubric

Use this rubric to judge `vgc-meta-research` outputs by competitive value first and efficiency second.

## Priority Order

Judge in this order:

1. `Format correctness`
2. `Source discipline`
3. `Freshness handling`
4. `Inference honesty`
5. `Meta usefulness`
6. `Anti-meta value`
7. `Efficiency`

A response that is cheap but wrong fails. A response that is expensive but current, trustworthy, and practically useful may still pass.

## Primary Quality Dimensions

### `Format correctness`

The response:

- confirms the active regulation accurately
- states the active mechanics accurately
- does not carry over stale Scarlet/Violet assumptions as current Champions facts
- corrects user misunderstandings when they conflict with the live format

### `Source discipline`

The response:

- uses official sources first for legality, rules, dates, and mechanics
- uses current community trackers for usage, cores, and trend claims
- does not blur official rules with community inference
- cites enough sources for the reader to check the answer

### `Freshness handling`

The response:

- treats current meta claims as time-sensitive
- indicates when data is early, thin, or conflicting
- prefers live verification for present-tense claims
- avoids presenting older snapshots as if they were confirmed-current

### `Inference honesty`

The response:

- separates sourced facts from interpretation
- labels weak-data conclusions as inference
- avoids fake precision or unsupported certainty
- narrows claims when the evidence base is thin

### `Meta usefulness`

The response:

- identifies the real top threats and cores a player should prepare for
- explains the strategic shape of the field, not just a pile of usage names
- makes the information actionable for team prep or tournament decisions
- avoids generic “goodstuff” commentary with no competitive signal

### `Anti-meta value`

The response:

- offers concrete anti-meta openings or prep angles
- ties those openings to actual meta conditions
- explains why the recommended opening works
- avoids vague advice like “use something unexpected”

## Secondary Optimization Signals

### `Efficiency`

Use only after the quality dimensions above.

The response:

- avoids needless repetition
- loads only the references needed for the question
- gets to a useful answer without waste

High token use is only a real problem if:

- it does not improve answer quality, or
- it makes the workflow too slow or costly for real use

## Pass / Fail Gates

Fail the response if any of these happen:

- inactive mechanics are presented as active
- stale SV-era assumptions are presented as current Champions truth
- invented usage figures or unsupported certainty appear
- official rules and community inference are conflated
- the answer is too vague for a competitive player to use

## Suggested Human Scorecard

Use a simple 0-2 score for each primary quality dimension:

- `0` = failed or missing
- `1` = partial / mixed
- `2` = strong

Treat `Efficiency` as a note, not a deciding score, unless the output quality is otherwise comparable.

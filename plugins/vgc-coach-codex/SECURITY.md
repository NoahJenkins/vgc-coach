# Security Policy

## Supported Versions

Security fixes are applied on the current `main` branch.

## Reporting A Vulnerability

Do not open a public issue for a suspected vulnerability.

If GitHub security advisories are enabled for this repository, use the private vulnerability reporting flow there. If that flow is not available yet, contact the maintainer through the repository owner's GitHub profile and request a private reporting channel.

Include:

- a concise description of the issue
- affected files, commands, or workflows
- reproduction steps or proof of concept
- impact and any suggested mitigation

You should receive an acknowledgement within a reasonable time after the report is seen. Public disclosure should wait until the issue has been reviewed and a fix or mitigation path is in place.
